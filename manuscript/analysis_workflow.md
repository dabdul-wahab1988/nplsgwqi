# Analysis Workflow (overview)

This workflow summarises how `data.csv` is transformed into the manuscript artifacts referenced in `manuscript/sections/**/*.md`.

```mermaid
flowchart TD
  A["Input data (`data.csv`)"] --> B["Preprocess: unit conversions (mg/L → meq/L), QA audit (CBE)"]
  B --> C["NPLS-GWQI + neutrosophic memberships (T/I/F)"]
  B --> D["Compositional discovery: ILR transform → PCA → CLR back-interpretation"]
  B --> E["Deterministic indices: Gibbs, CAI, Na/Cl, weathering proxies"]
  B --> F["Bayesian endpoint-driver models (horseshoe regression)"]
  B --> G["Health risk (deterministic + Bayesian)"]
  B --> H["Irrigation indices + N-ISI (entropy weighting)"]

  E --> I["PHREEQC saturation indices (SI)"]
  C --> J["Benchmarking vs conventional/fuzzy WQI + weight Monte Carlo"]
  F --> K["Diagnostics: PPC, PSIS-LOO/WAIC, prior sensitivity"]
  D --> L["SBP/basis sensitivity checks"]

  I --> Z["Artifacts (`manuscript/artifacts/*`)"]
  J --> Z
  K --> Z
  L --> Z
  C --> Z
  D --> Z
  E --> Z
  G --> Z
  H --> Z
```

## Reproducible artifact generation

The reviewer-response artifacts introduced in this revision are produced by:

- `manuscript/reviewer_revision_artifacts.py` (CBE tables, PHREEQC SI, Bayesian diagnostics/PPC, prior sensitivity, ILR basis sensitivity, WQI benchmarking, irrigation disagreement)
- `manuscript/reviewer_revision_artifacts_stage2.py` (risk-uncertainty decomposition and any additional risk summaries)

The baseline manuscript artifacts are generated by the project’s existing analysis workflow (see `generation_script.py` and the contents of `manuscript/artifacts/`).
